# Distroless Images 

- Distroless images are container images that contain only your application and its runtime dependencies — nothing else. No shell, no package manager, no bash, no apt, no unnecessary OS utilities — absolutely nothing that the application does not need to run.

- The name 'Distroless' means 'without a distribution' — meaning there is no full Linux distribution like Ubuntu or Debian inside the image. A normal base image like Ubuntu comes with hundreds of tools — ls, curl, wget, bash, apt, and many more. A distroless image strips all of that out and keeps only what is absolutely required to run your specific application.

- The biggest benefit is security. In a traditional container, if an attacker breaks into your container, they have access to a shell, package managers, and system tools — they can explore, install malicious software, and move laterally. In a distroless container, there is no shell to drop into. The attacker has nowhere to go. The attack surface is reduced to almost zero.

- The second benefit is size. Because there are no unnecessary tools or libraries, distroless images are significantly smaller than traditional base images — leading to faster pulls, faster deployments, and lower storage costs.

- Distroless images are maintained by Google and are available on GitHub and also on Google Container Registry.

# How to Use Distroless in Dockerfile

**For Example : Go Application — Smallest Possible Image**

dockerfile# Stage 1 — Build
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 go build -o myapp .

**Stage 2 — Distroless (no shell, nothing)**

FROM gcr.io/distroless/static
COPY --from=builder /app/myapp /myapp
ENTRYPOINT ["/myapp"]

Image size:
  golang:1.21 single stage  →  800 MB
  distroless/static          →  ~5 MB

**Key Notes**

Distroless images are commonly used in production environments because they:

  - Improve security

  - Reduce vulnerabilities

  - Decrease image size

  - Improve container startup performance
